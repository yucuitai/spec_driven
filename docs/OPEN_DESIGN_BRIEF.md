# Open Design Brief — Spec-Driven 工具

> 这份文档是给 Open Design / 任意 AI 设计 Agent 的设计需求书。
> 把它整体粘进 Open Design 的 brief 输入框，或在 Open Design 的项目里另存为 `brief.md`，再选一个合适的 Skill（推荐 Web Prototype）即可生成原型。

---

## 1. 产品定位（一句话）

一个引导用户走完 **Spec-Driven 开发全流程** 的 Web 工具：从想法到 Phase 拆解到 AGENTS.md 生成，一站式搞定。

**理念**：用户负责想清楚要什么，AI 负责实现。本工具帮用户把"想清楚"这件事结构化。

## 2. 目标用户

- 想用 AI 编程但不知道怎么开始的开发者
- 独立开发者、Vibe Coding 新手
- 像作者本人一样刚毕业的初级开发者

> 注意：用户主要是工程师，不是设计师，**界面应当是工具感、安静、利落**，不要做营销味重的 SaaS 落地页。

## 3. 关键场景（用户旅程）

```
打开首页（30 秒理解这是什么） →
进入向导（4 步表单，旁边有 AI 助手实时给建议） →
预览 Spec（Markdown 渲染，可编辑，可让 AI 重写整篇） →
预览 Phases（同上） →
预览 AGENTS.md（同上） →
导出页（三个文档可分别下载或打包 zip）
```

中途用户可随时进 **设置页** 配置 AI（OpenAI 兼容协议、API Key 存 localStorage）。

## 4. 信息架构与页面清单

需要为下面 7 个页面各出一版设计：

| 页面 | 路径 | 用途 | 关键内容 |
|---|---|---|---|
| 首页 | `/` | 介绍产品、引导开始 | hero 区、4 步流程图、5 个核心功能卡片、CTA 按钮 |
| 向导页 | `/wizard` | 4 步表单 + 右侧 AI 助手 | 步骤进度条、当前步骤的表单、AI 助手侧栏（可流式输出） |
| Spec 预览页 | `/preview/spec` | Markdown 渲染 + 编辑 + AI 重写 | 大块文档卡片、编辑/预览切换、底部 AI 操作面板 |
| Phases 预览页 | `/preview/phases` | 同上 | 同上 |
| AGENTS 预览页 | `/preview/agents` | 同上 | 同上 |
| 导出页 | `/export` | 三个文档预览卡片 + 下载 | 三张卡片并排、各自下载 + 打包下载 |
| 设置页 | `/settings` | 配置 AI 凭证 | 三字段表单（Key / BaseURL / Model）、测试连接按钮、清除按钮 |

**全局**：每页顶部都有一条 sticky 的 Header（品牌 logo + 主导航 + 右上角设置入口 + AI 配置状态指示灯）。

## 5. 全局导航

- 主导航（横排）：首页 / 向导 / Spec / Phases / AGENTS / 导出
- 右上角：设置（带一个绿点/灰点表示 AI 是否已配置）

## 6. 品牌与视觉调性

- **气质**：开发者工具感，干净、安静、有秩序、略带技术感的克制
- **绝对不要**：营销味、SaaS 拼贴感、装饰性渐变 orb、磨砂玻璃、紫粉撞色
- **字体**：英文 Inter / 数字与代码 JetBrains Mono；中文用系统默认无衬线，注意行高 1.6 以上
- **圆角**：8px 为主，按钮 6px，输入框 6px，卡片 8-10px；不要"超大圆角药丸"
- **阴影**：极轻或不用，靠 1px border 分割
- **配色建议**：以中性灰为主体（near-black 文本 + 浅灰边框 + 白底），点缀色用一个偏冷的强调色（如 `oklch(0.5 0.15 250)` 这种克制的蓝紫），危险色用克制的红
- **支持暗色模式**（必须），暗色应当是真正的工程师暗色：深灰背景（不要纯黑）+ 略低饱和的强调色

## 7. 关键组件规范

- **Header**：56px 高，sticky，半透明白底 + backdrop-blur，下边一根 1px 分割线
- **按钮**：三个层级（primary 实心 / outline / ghost），高度 36–40px，icon 跟文字间距 6px
- **卡片**：1px border + 8px 圆角 + 16-24px 内边距，hover 不要放大、最多变色边
- **输入框**：和按钮等高，focus 时换 border 颜色而不是加粗
- **Markdown 渲染区**：标题分级清晰、代码块有底色、表格有横线、待办列表的 `[ ]` 渲染成 checkbox
- **AI 助手面板**（重要）：右侧侧栏，宽 360px，包含：
  - 顶部标题 + 当前步骤名 + Sparkles 图标
  - 中间：未生成时显示一句引导语 + 「让 AI 给建议」按钮
  - 生成中：显示 loading + 流式输出 + 「停止」按钮
  - 生成后：Markdown 渲染结果 + 「采纳」按钮（如果适用）+ 「重新生成」按钮
  - 未配置 API Key 时：按钮禁用，显示 "前往设置配置 API Key" 链接

## 8. 关键页面的具体重点

### 首页
- Hero 区一定要"工具感"：左上是品牌名（小，sans-serif，不堆装饰），中间一句话描述（"从产品描述到完整项目文档，一站式搞定"），下面两个按钮（开始创建 / 配置 AI）
- 4 步流程图横排（数字圆 + 文字 + 箭头），不要做成华丽 timeline
- 5 个核心功能用 card grid（2 列 / 3 列响应式），每张卡片：左上 lucide 图标 + 标题 + 一句话描述
- 不要 footer 堆社交链接，只一行版权 + GitHub 链接

### 向导页（最重要的设计点）
- 顶部步骤进度条（4 步、当前高亮、过去置灰、未来更淡）
- 主区分两栏：**左 60% 当前步骤的表单 / 右 40% AI 助手面板**
- 移动端两栏堆叠，AI 助手面板可折叠为底部抽屉
- 表单字段间距要足够（垂直 24px+），不要堆得太紧
- 上一步 / 下一步按钮固定在主区底部

### 预览页（Spec / Phases / AGENTS 三页同构）
- 一个大块文档卡片（占主视觉），顶部带"编辑/预览/保存"按钮组
- 卡片下方一个 AI 操作面板（"让 AI 重写整篇" + 流式输出区 + 应用/丢弃按钮）
- 最底部一对返回 / 下一步按钮
- Markdown 渲染要漂亮：参考 GitHub README 风格，但更安静

### 设置页
- 单卡片表单，最大宽 600-680px 居中
- 三个字段（Key / BaseURL / Model），每个下面附一行小灰字说明（举例值）
- "测试连接并保存" 按钮 primary，"清除" 按钮 ghost + destructive 颜色
- 顶部要有一段醒目但不张扬的告知："**所有信息仅保存在你的浏览器（localStorage），不会上传任何服务器**"

## 9. 现状（参考用，不是约束）

当前已用：Next.js 15 + React 19 + Tailwind CSS 4 + zustand + 一套 shadcn/ui 风格的本地 UI 组件（Button / Card / Input / Textarea / Badge）+ react-markdown + remark-gfm + lucide-react 图标。

**设计可以推翻当前外观**，但要确保产物能落到上述技术栈里（也就是说：用 Tailwind utility class + CSS 变量主题、组件 API 类似 shadcn 风格即可，不要依赖 Figma 专有效果或外部插件）。

## 10. 输出诉求

请以 **Web Prototype（单页 HTML 真组件）** 的形式产出 7 个页面，附带：

1. 一份完整的 `DESIGN.md`（颜色 Token / 字号阶梯 / 间距阶梯 / 组件规则 / 暗色映射）
2. 每页一张可交互的 HTML 原型
3. 移动端、桌面端各一版
4. 关键交互状态：按钮 hover/active/disabled、表单 focus/error、AI 面板 idle/loading/done/error

## 11. 不要做的事

- 不要做 hero 大图 / 渐变 orb / 大块插画
- 不要把 H1 写成营销口号；它就是工具名 + 一句话描述
- 不要把页面区块全部塞进卡片里嵌套（"卡套卡"是禁忌）
- 不要紫粉撞色
- 不要做带圆角的"巨型 CTA 横幅"
- 不要把 AI 助手面板做成聊天框气泡（它是侧边任务面板，不是对话框）
