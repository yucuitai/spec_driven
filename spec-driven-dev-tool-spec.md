# Spec-Driven 开发全流程工具

## 产品定义

**一句话**：一个引导用户走完 Spec-Driven 开发全流程的 Web 工具，从想法到 Phase 拆解到 AGENTS.md 生成，一站式搞定。

**给谁用**：想用 AI 编程但不知道怎么开始的开发者、独立开发者、Vibe Coding 新手、像我这种刚毕业的初级开发者。

**核心理念**：你负责想清楚要什么，AI 负责实现。这个工具帮你把"想清楚"这件事结构化。

---

## 技术栈

| 层面 | 选择 | 理由 |
|---|---|---|
| 框架 | Next.js 15 | 前后端一体，开发快，部署方便 |
| UI | Tailwind CSS + shadcn/ui | 组件现成，风格专业，AI 生成代码质量高 |
| 状态管理 | Zustand | 轻量，适合多步骤表单场景 |
| 存储 | LocalStorage 先行 | MVP 不需要数据库，后期再加 |
| 部署 | Vercel | Next.js 原生支持，零配置上线 |

**简记**：Next.js + Tailwind + shadcn/ui，先纯前端跑通，后期按需加后端。

## 核心功能

### 1. 产品定义向导
- 引导用户一步步填写产品信息
- 必填项：产品名称、一句话描述、目标用户、核心功能（3-5 个）
- 可选项：参考产品、技术偏好、约束条件

### 2. Spec 生成
- 根据用户输入自动生成结构化的 Spec 文档
- 包含：产品概述、功能列表、页面/视图、数据模型、用户流程
- 用户可在线编辑和调整生成结果

### 3. Phase 拆解
- 把 Spec 自动拆分成可执行的开发阶段
- 每个 Phase 包含：标题、描述、验收标准、预估复杂度
- 拆解原则：每个 Phase 结束时产品可运行，先骨架后细节

### 4. AGENTS.md 生成
- 根据用户选择的技术栈生成项目指令文件
- 包含：技术栈说明、代码风格、构建命令、项目结构、硬性约束
- 可自定义补充额外指令

### 5. 一键导出
- 下载 Spec 文档（Markdown）
- 下载 Phase 计划（Markdown）
- 下载 AGENTS.md
- 可选：打包下载全部文件

## 页面/视图

| 页面 | 说明 |
|---|---|
| 首页 | 产品介绍 + 开始按钮 |
| 向导页 | 多步骤表单，引导填写产品信息 |
| Spec 预览页 | 展示生成的 Spec，支持编辑 |
| Phase 计划页 | 展示拆解的 Phase 列表，支持调整顺序和编辑 |
| AGENTS.md 配置页 | 选择技术栈 + 预览生成的 AGENTS.md |
| 导出页 | 预览所有文档 + 下载 |

## 用户流程

```
打开首页
  ↓
点击"开始创建"
  ↓
进入向导，逐步填写产品信息（4-5 步）
  ↓
生成 Spec → 预览/编辑
  ↓
自动拆解 Phase → 预览/调整
  ↓
配置技术栈 → 生成 AGENTS.md
  ↓
一键导出所有文档
```

## 开发 Phase 拆解

| Phase | 内容 | 验收标准 |
|---|---|---|
| Phase 1 | 项目骨架：初始化 Next.js + Tailwind + shadcn/ui，跑通空页面 | `npm run dev` 能看到首页 |
| Phase 2 | 向导组件：多步骤表单，能输入产品信息并存储到 Zustand | 填完所有步骤后数据在状态中 |
| Phase 3 | Spec 生成：根据输入生成结构化 Spec 文档 | 填完向导后能看到完整 Spec |
| Phase 4 | Phase 拆解：根据 Spec 自动生成开发阶段列表 | Spec 旁边能看到 Phase 列表 |
| Phase 5 | AGENTS.md 生成：技术栈选择 + 文件预览 | 选择技术栈后能看到生成的 AGENTS.md |
| Phase 6 | 导出功能：Markdown 下载 + 打包下载 | 点击下载能得到完整文件 |
| Phase 7 | UI 打磨 + 响应式 + 暗色模式 | 手机和桌面端体验流畅 |
| Phase 8 | 部署上线 | Vercel 部署后可公开访问 |

## AGENTS.md 模板（供产品内置使用）

当用户选择不同技术栈时，产品应能生成对应版本的 AGENTS.md。以下是通用模板结构：

```markdown
# 全局配置

- 语言偏好：[中文/英文]
- 代码风格：[具体规范]

# 技术栈

- 框架：[Next.js / Vue / ...]
- UI：[Tailwind / ...]
- 状态管理：[Zustand / ...]
- 数据库：[无 / PostgreSQL / ...]

# 项目结构

[目录结构说明]

# 命令

- 启动：`npm run dev`
- 构建：`npm run build`
- 测试：`npm test`

# 约束

[用户自定义约束]
```

## 备注

- MVP 阶段不加用户登录，所有数据存 LocalStorage
- 后期可加：用户账号、云端存储、Spec 协作编辑、AI 对话式调整
- Spec 生成可以用简单的模板拼接，不需要调用 LLM API（MVP 阶段）
